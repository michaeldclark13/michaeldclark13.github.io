D3 Visualization

For my first visualization, I used the data-driven document tool (D3) to display the the number of male and female patients that were were studied over the roughly 5000 day study. The X-axis for each graph represents the number of days before or after the 1st TBI.

D3 is an incredibly verstile tool since it harnesses the power of Javascript code from which an almost limitless amount of visualizations can be generated. D3 can draw lines, points, rectangles (bar graphs), and circles using svg, and there are many APIs available to render more advanced visualization techniques. All data types also are supported. The one major downside to D3 is the amount of time and knowledge that must first be invested before any meaningful visualizations can be rendered. A strong knowledge of Javascript is also necessary to refine the data prior to visualization. The convention of stringing together many .function() statements is also a confusing and frankly poorly designed framework for readable code.

I would only use D3 as a visualization tool if my data required an high level of customization and control. Using D3, arrays of data can be parsed via Javascript and then used to draw a variety of shapes and diagrams, all without having to modify a CSV file beforehand.

2nd NOTE: The gender which is currently displayed can be changed by pressing the button below the visualization.

Tableau Visualization

For my second visualization, I used the Tableau Public tool to display the occurrance of a variety of symptoms (PTSD, Sleep Difficulties, Alzheimers, Depression, Endocrine Issues, and Stress) over the 9 year study across the different age groups. Overall, a high number of depression cases were reported for ages 18-34 and a high number of sleep and PTSD disorders were reported for ages 25-34.

Tableau public is quite an easy tool to use because of its powerful and yet simple user interface. The tool allows for quick import of excel data from which data can be modified and finally accepted. These columns are broken into different dimensions that can be used in a variety of ways to create bar graphs, scatterplots, line graphs, and more advanced visualizations. Because of its ease of use, a high number of visualizations can be rendered from one or more data sets at a time. Once the user is happy with the data sets, he or she can publish her visualizations online and then embed the interactive visualization into an html page (as seen below). Most of the common data types are supported by tableau, while volume rendering is not a supported feature of the interface.

I would use Tableau public in the future because of its ease of use and its powerfulful interface. The one downside of this tool in comparison to D3 would be its lack of programmability. Code offers an almost endless number of possibilities in terms of manipulation and visualization of data. Regardless of the features offered by the Tableau interface, if a data set required a high level of customability, a different visualization tool should be utilized.
